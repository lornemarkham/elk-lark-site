# How Charlie & Lorne Work

## Principles

- Build fast, not perfect
- Always prioritize flow + functionality over polish
- One step at a time — no overengineering
- Keep everything Phase-based

## Working Style

Every step should include:

1. What we’re building
2. Why it matters
3. What Lorne should know
4. Clear action (prompt or task)
5. What success looks like

## Product Mindset

- We are building loops, not screens
- Every feature should connect to something else
- Always think: "What does the user feel after this action?"

## Current Phase

Phase 1 — UI + Flow (local-first, mocked)

## Rules

- No backend yet
- No overbuilding
- No premature optimization
